# 语言模型基线性能测试指南 (Baseline Benchmark)

本目录下的工具用于自动化执行 LLM（大语言模型）的推理性能基线测试，默认基于 SGLang 框架。它能够根据预设的负载矩阵自动运行测试、追踪进度并生成可直接导入 Excel 的结构化性能分析报告。

## 1. 配置文件说明 (YAML Config)

测试负载和服务器配置均推荐通过 YAML 文件（例如 `configs/workloads/optimal_sets.yaml`）进行集中管理。

YAML 文件主要包含两部分：
*   **`server_config`**: 服务端运行参数配置。
    *   `model_path`: 模型权重的本地路径。
    *   `tp`: 张量并行度 (Tensor Parallelism) 大小。
    *   `host` / `port`: 绑定的 IP 地址和端口（例如使用 `0.0.0.0:30001`）。
    *   `extra_args`: 引擎启动时的其他附加配置（如开启 FlashInfer、分配显存比例等）。
*   **`sets`**: 测试用例矩阵（黄金矩阵），负责按短周期 (Short)、常规 (Normal)、纯预填充 (Prefill Only) 和纯解码 (Decode Only) 结构列出具体的任务分组。
    *   `io_pairs`: 输入 / 输出 Token 长度的组合。
    *   `concurrency_and_prompts`: 压测并发数和总的请求 Prompt 数量组合。

## 2. 运行基线测试 (Benchmarking Modes)

你可以通过执行 `./run_benchmark.sh`，或者直接运行 `python run_eval.py --workload <yaml_path>` 来启动测试。底层脚本具备端口探活能力，支持以下两种工作模式：

### 模式一：一体化模式 (Deploy and Bench / 自动部署并测试)
在此模式下，你不需要手动启动任何模型推理服务：
*   脚本会首先检测 YAML 中（或命令行指定的） `host:port`。
*   如果发现该端口**未被占用**，脚本将在本地根据 `server_config` 自动拉起 SGLang 推理服务进程。
*   等待服务就绪后，脚本会自动打满所有的压力测试组合，并且在测试完成或遇到异常退出后，**自动安全关闭**所起的推理服务。
*   *适用场景：需要在单机上一键完成“跑分并生成报告”的日常自动化测验。*

### 模式二：分离模式 (Deploy then Bench / 先部署后测试)
适用于远端服务器或者你需要手动控制、常驻服务端配置的场景：
*   **第一步 (Deploy)**：在目标机器或容器上手动启动 SGLang 推理服务（必须监听配置中指定的同样的 `host` 和 `port`）。
*   **第二步 (Bench)**：运行本评测脚本。脚本通过 Socket 探测到该端口已有服务存活，便直接进入 `[HYBRID MODE] Separated` 模式。
*   在此模式下，本脚本仅作为发压客户端运作，依次发送所有评测请求评测，测试完成后**不会**强行关闭你的远端服务端进程。
*   *适用场景：跨机器测试、远程评测、调试服务参数或服务在 Kubernetes 容器里作为常驻服务的情况。*

## 3. 测试结果输出 (Output)

测试开始后，控制台会显示直观的全局进度条 `Overall Benchmark Progress`，并提示当前所跑任务的状态。
所有结果将生成在 `results/` 目录下（按“时间戳_模型名_tp参数_运行模式”命名文件夹），主要留存以下文件：
1.  **`sgl_bench_*.csv`**: 压测工具原生产生的实时结果表格（带有原始过程数据）。
2.  **`elapsed_time.json`**: 记录了各组测试任务消耗时长的精细时序度量，用于评估负载耗时。
3.  **`env_info.json`**: 记录集群当前的软硬件环境信息（如显卡型号、驱动版本、操作系统变量）。
4.  **`final_report_*.csv`**: 最终生成的结构化基线报告！该表格自动清洗了冗余数据、剔除了 ID 干扰并填齐了所有关联信息，**可直接全选复制粘贴进 Excel 中展示**。
